## 🎫 Sistema de tickets

OBRIXA utiliza **GitHub Issues** como sistema de gestión de tareas e incidencias durante el desarrollo del proyecto.

Este sistema permite organizar el trabajo del equipo, hacer seguimiento del progreso y mantener un historial claro de los cambios realizados en la aplicación.

Cada funcionalidad o problema detectado se gestiona mediante un **Issue**, que puede estar asociado a una rama de desarrollo y posteriormente a un **Pull Request**.

### Tipos de Issues

El proyecto utiliza los siguientes tipos de incidencias:

🔹 **feature**  
Nuevas funcionalidades que se incorporarán a la aplicación.

Ejemplos:
- Crear visor de plano interactivo
- Implementar creación de incidencias sobre el plano
- Añadir asignación de tareas a operarios

🔹 **bug**  
Errores detectados en el funcionamiento de la aplicación.

Ejemplos:
- Error al cargar el plano de la obra
- Crash al abrir el detalle de una incidencia
- Problema al guardar un ticket

🔹 **improvement**  
Mejoras en funcionalidades existentes o en la arquitectura del proyecto.

Ejemplos:
- Optimizar carga de planos
- Mejorar diseño de la pantalla de incidencias
- Refactorizar repositorios de datos

### Flujo de trabajo

El flujo de trabajo del proyecto sigue estos pasos:

1. Se crea un **Issue** describiendo la tarea o problema.
2. Se crea una **rama de desarrollo** asociada a ese issue.
3. Se implementa la solución o funcionalidad.
4. Se abre un **Pull Request**.
5. Tras revisión, se integra en la rama principal del proyecto.

Esto permite mantener un desarrollo **organizado, trazable y colaborativo**.

## 👨‍💻 Autor

Proyecto desarrollado como parte de un trabajo académico enfocado en el desarrollo de una **aplicación Android para la gestión digital de incidencias en obras de construcción mediante planos interactivos**.

La aplicación busca mejorar la coordinación entre encargados y operarios, permitiendo localizar y gestionar incidencias directamente sobre los planos de la obra.

## 📱 Icono de la aplicación

El icono de **OBRIXA** representa el concepto central de la aplicación: la gestión de incidencias sobre planos de obra.

El diseño combina tres elementos visuales principales:

- 📐 **Plano técnico** → representa la base visual sobre la que se registran incidencias.
- 📍 **Marcador de ubicación** → indica el punto exacto donde ocurre una incidencia en la obra.
- 🏗️ **Colores de construcción** → azul tecnológico y naranja industrial.

Este diseño refleja la idea de una herramienta digital que permite **localizar, gestionar y resolver incidencias directamente desde el plano de una obra**.
